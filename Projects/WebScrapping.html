Data Automation & Trade Compliance Project

HS Code Trade Compliance Data Extraction System

Designed and developed an automated data pipeline that retrieves tariff rates and import licensing requirements from Indonesia's National Single Window (INSW) portal. The solution eliminates repetitive manual lookups by collecting, structuring, and exporting regulatory data into Excel and JSON formats for operational reporting and analysis.

PROJECT OVERVIEW

At work, HS Code tariff and licensing information often needed to be checked manually through the INSW portal. When processing dozens or hundreds of products, this repetitive task became time-consuming and prone to human error.

To improve efficiency, I developed a Python-based automation tool that automatically retrieves tariff and licensing data, structures it into Excel, and generates JSON outputs for reporting and analysis.


This project automates the entire process by identifying incomplete records, retrieving regulatory information directly from the INSW portal, transforming unstructured content into structured datasets, and exporting the results into Excel and JSON files for downstream analysis.

TECH STACK

  • Python
  • Playwright (Browser Automation)
  • Regular Expressions (Pattern Extraction)
  • OpenPyXL (Excel Processing)
  • JSON Serialization

DATA EXTRACTED

The automation pipeline captures and structures the following regulatory information:


  • HS Code
  • BM MFN (Import Duty Rate)
  • PPN (VAT)
  • PPH API
  • PPH Non-API
  • Import Licensing Requirements
  • Customs Permit Codes

TECHNICAL CHALLENGES & SOLUTIONS

The INSW portal does not provide a public API for accessing the required trade compliance data. To overcome this limitation, browser automation was implemented using Playwright to replicate user interactions such as searching HS Codes, opening tariff details, and navigating import regulation pages.


Regulatory information was embedded within unstructured page content rather than exposed through structured endpoints. Regular expressions were used to identify and extract tariff percentages, tax rates, permit names, and customs permit codes into a standardized format.


To improve processing efficiency, the system automatically detects incomplete records and only processes missing HS Codes rather than reprocessing the entire dataset.
